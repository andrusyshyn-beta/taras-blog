# Чеклист “Готовності сайту” під 2026: Google Search + AI Overviews + LLM-краулери

## 1) Технічний фундамент (індексація, доступність, швидкість)
- [ ] robots.txt не блокує важливі сторінки та не блокує AI-ботів без потреби
- [ ] sitemap.xml існує, валідний, оновлюється, містить canonical-URL (без сміття / параметрів)
- [ ] HTTP статуси: ключові сторінки повертають 200, немає масових 3xx/4xx/5xx
- [ ] Canonical: на сторінках стоїть правильний canonical, без циклів і конфліктів
- [ ] Meta robots: немає випадкових noindex на важливих сторінках (особливо blog/pillar/pricing)
- [ ] JS-залежність: основний контент читається без виконання JS (або хоча б server-rendered)
- [ ] Core Web Vitals / швидкість: сторінки не “важкі”, LCP/INP/CLS у нормі (особливо мобільно)
- [ ] Кешування/CDN налаштовані, великі файли (зображення) оптимізовані
- [ ] Мобільна придатність: текст читається, кнопки клікабельні, немає “layout shifts”
- [ ] Логи/аналітика: доступні server logs або хоча б повні події в аналітиці для діагностики ботів/індексації

## 2) Структура сайту та навігація (щоб Google і LLM “бачили карту”)
- [ ] Є чіткі хаби/піллари (навіть 5–7) і до них ведуть внутрішні лінки
- [ ] Блог не “плоский”: кожна стаття має мінімум 3–4 внутрішні лінки (pillar + siblings + adjacent)
- [ ] Немає “сиріт” (orphan pages): кожна важлива сторінка має хоча б 1 вхідний внутрішній лінк
- [ ] Breadcrumbs (якщо є) — коректні
- [ ] URL-структура послідовна (без дублювань через / та без хаосу зі слугами)

## 3) Контент-готовність під SEO + GEO/AEO
- [ ] Кожна стаття має 1 primary keyword і 3–6 secondary (природно, без набивання)
- [ ] В кожній статті є TL;DR на початку (2–3 буліти) — для LLM/AI Overviews
- [ ] H2 часто оформлені як питання, а перше речення під H2 — пряма відповідь
- [ ] Є FAQ 3–5 Q/A (коротко, по справі)
- [ ] Є приклади/шаблони (copy/paste), а не лише “пояснення”
- [ ] Є 2 нативні CTA-вставки в тілі (не тільки в кінці), без “продажного крику”
- [ ] Немає непідтверджених “фактів/цифр” (особливо про ефективність/конверсії/ринок)
- [ ] Є “Last updated” (особливо для pillar-гайдів) і план оновлення раз на 3–6 місяців

## 4) Schema та “машиночитність” (must-have)
- [ ] На статтях є Article/BlogPosting schema
- [ ] На FAQ-сегменті (якщо він є) — FAQPage schema
- [ ] Якщо є інструкції/чеклисти — HowTo schema (де доречно)
- [ ] На сайті є Organization schema + sameAs на ключові профілі (LinkedIn, YouTube, X, etc.)
- [ ] OpenGraph/Twitter cards налаштовані (щоб шерінг давав гарний прев’ю)

## 5) Довіра / Entity Authority (мінімум для B2B)
- [ ] Є сторінка About з чітким описом сутності (хто ви, що робите)
- [ ] Є контакти (email/форма), прозорість компанії/автора
- [ ] Є сторінки Privacy / Terms / Cookie (і вони не виглядають як “заглушки”)
- [ ] Авторство/редакційність: на статтях вказано автора (або редакційний бренд)
- [ ] Узгодженість назви/опису продукту на всіх профілях (одна сутність, одна позиція)

## 6) Конверсія (щоб трафік не був “в нікуди”)
- [ ] Є зрозумілий шлях: Blog → Pillar → Product page → Signup/Demo
- [ ] CTA не один: є м’який (підписка/демо) і жорсткіший (signup)
- [ ] На сторінках продукту є: “для кого”, “що вирішує”, “як працює”, “що отримає керівник”
- [ ] Форми/кнопки працюють, немає зайвих кроків, сторінка “дякую” існує

## 7) Вимірювання та контроль (без цього немає керування)
- [ ] Підключено Google Search Console
- [ ] Налаштовано аналітику (GA4 / PostHog / Plausible) + ключові події (signup, demo, etc.)
- [ ] Є дашборд “index coverage”: скільки сторінок в sitemap vs скільки в індексі
- [ ] Є дашборд “content quality signals”: bounce/engagement по кластерам
- [ ] Є контроль 404/redirect-ланцюгів

## 8) “Гігієна” для AI-трафіку (без фанатизму, але правильно)
- [ ] Контент доступний без paywall і без важкого JS
- [ ] Сторінки мають чіткі блоки 75–300 слів (самодостатні секції)
- [ ] На важливих сторінках є “джерела/посилання” там, де згадуються факти
- [ ] (Опційно) llms.txt додано як підказка
- [ ] Є RSS-фід для блогу: /rss.xml або /blog/rss.xml
- [ ] Фід валідний (правильний XML, дати, енкодинг), не віддає 4xx/5xx
- [ ] У <head> сторінок блогу є discovery-лінк (`rel="alternate" type="application/rss+xml"`)
- [ ] Фід містить: title, link, description, кожен item має guid, link, pubDate, summary
- [ ] RSS-URL додано в llms.txt як “recommended sources”

## 9) Мультимовність / Регіональність та Sitemap-гігієна
- [ ] hreflang налаштований між мовними версіями
- [ ] Локалізовані сторінки мають правильні canonical
- [ ] Сторінки тегів/категорій/архівів або якісні, або під noindex,follow
- [ ] sitemap.xml має lastmod, не містить 404, дублів чи редіректів
- [ ] WebSite + SearchAction schema
- [ ] BreadcrumbList schema
- [ ] Один “канонічний” варіант домену (https, www/non-www)
- [ ] Немає mixed content (http-ресурси на https-сторінках)
