5 днів тому Anthropic злила вихідний код Claude Code. Випадково. 512,000 рядків. Повна внутрішня архітектура продукту на $2.5 мільярди річного доходу — просто лежала у відкритому доступі.

Дослідник безпеки Chaofan Shou помітив це вранці і кинув посилання в X. Далі GitHub зробив своє — за кілька годин тисячі форків. Anthropic запустила 8,000 DMCA-запитів на видалення. Поки вони це робили, корейсько-канадський розробник Sigrid Jin — та сама людина яка витратила 25 мільярдів токенів Claude Code за рік — встигла переписати весь код на Python до сходу сонця. Без DMCA на неї не підеш.

Це вже друга подібна штука за тиждень. За п'ять днів до цього злились 3,000 внутрішніх файлів включно з деталями нерелізованої моделі «Mythos». Два рази за тиждень. Компанія на $380 мільярдів оцінки, яка готує IPO на $60 мільярдів восени.

Anthropic весь час позиціонує себе як "безпечна AI-компанія". Constitutional AI, відповідальний розвиток, ми не OpenAI. І одночасно двічі за тиждень роняють операційку як стартап на першому місяці.

Я не кажу що вони погана компанія — Claude Code реально крутий продукт, я сам користуюсь. Але є щось показове в тому що компанія яка найбільше говорить про безпеку AI не може утримати власний вихідний код від потрапляння в публічний доступ через "помилку пакування".

Конкуренти — OpenAI, Google, xAI — отримали детальну карту того що будували роками. Без реверс-інжинірингу. Безкоштовно.

Чи змінить це щось у реальному конкурентному балансі — питання відкрите. Але точно змінить розмову навколо IPO.
